XML Rules

XML has rules that must be followed in order to be a valid xml file. Describe 6 of these with code examples below.

An XML file must have a root element

Every XML document has exactly one root element which encloses all other elements within it, making it the parent to every element within the document.

Example:

<root>
 <child></child>
 <child></child>
 <child></child>
</root>

All XML elements must have closing tag

When a tag is declared, it must also be closed afterwards. This is unlike HTML, which allows some elemnts to have no end tag. Even empty tags must be closed for the XML document to be valid.

Example:

<correct></correct>
<incorrect>

Case sensitivity matters

Because XML is a case sensitive document, opening and closing tags must match exactly. Even though XML doesnt mind if there is uppercase characters in the tags (as long as they match), some applications using XML do, so it is important to check naming conventions beforehand, or just be safe and use all lowercase characters.

Example:

<correct></correct>
<inCorrect></incorrect>

Elements can have an attribute tag

Attribute values modify a tag or help identify the type of information being tagged, developers used to HTML can compare it to using a selector, like id or class.

Example:

<address category="work"></address>

White space is preserved

In an XML document, any extra spaces or line breaks will be read, and can cause issues with the document if not handled correctly.

Example:

<address category = "work">
<address    category = " work">


are two very different elements.

XML can include comments

Just like HTML, XML allows developers to comment within the file itself, making for great documentation and readability of the file. XML comments are used in exactly the same format as in HTML as well, making it easy to remember.

Example:

<!--This is an XML comment-->
